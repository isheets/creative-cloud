TWITTER EMOJI CLOUD

Full Stack Data Project

A group project that live-streamed data from the Twitter API and showed various visualizations.

Twitter Emoji Cloud is a full-stack project that I developed with three classmates for a course called Big Data Architecture.

We used Java to build an Apache Storm topology which processed raw data from the Twitter Sample API to extract emojis from tweets and insert that data into a Cloud SQL database. Most of my work was done in this area and when we completed the topology, I containerized it using Docker and deployed to Google Kubernetes Engine.

We used Express.js to write a RESTful API which exposed our Google Cloud SQL data to the front-end application. The API was deployed to Google App Engine.

Our front-end visualizations used a library called ZingChart to show which emojis were trending on Twitter realtime and historically.

This project is no longer hosted due to the hosting costs associated with Google Cloud Platform, but the code is fully accessible on GitHub.

Key Features

  • Full-stack
  • Google Cloud Platform Deployment
  • GitHub Collaboration
  • Data visualizations

CU SYLLABUS ARCHIVE

User Experience Design

A group user experience design consulting project for the CU Student Services Portfolio Manager

Throughout the Spring 2019 semester I worked with a team of two other students to research, design, and user test a new user experience for the CU Syllabus Archive for our client Jacie Moriyama, CU Student Services Portfolio Manager

My team's consoltation was necessary because the university wanted to provide a more valuable experience to students and faculty looking to view past syllabi for various courses.

We began our design process by analyzing the Syllabus Archive system that was currently in place and determining what changes would be useful. After this we created an initial prototype of the system in Adobe XD.

After prototyping, we interviewed a variety of CU faculty and students to get feedback on our design so that we could fine tune it to the needs of potential users.

We repeated the process of prototyping and user testing several times until we felt that our design addressed all the needs of the users and accomplished the initial design considerations we were provided with at the start of the project.

To finish the project, we distilled our findings into a design report and recorded a video demonstrating our XD prototype and delivered these to our client.

Key Features

  • User-centered design
  • Interactive feedback
  • Professional design report
  • Client collaboration

HABITS

iOS and Android Mobile Application

A multi-view application created for both iOS and Android devices.

Habits is a fully functional application that I created for both iOS and Android devices. The app is geared towards users that have undesirable habits that cost them money.

Users can enter the details for any sort of habit and define how regularly they engage in it and what it costs them. The app then displays how long it has been since the habit has been indulged and how much money they have "saved" as a result. The user can self report when they indulge the habit to reset the "money saved" and "time since" data. Details about the habit can be changed at any time.

Both the iOS and Android versions are nearly identical functionality that includes multiple views/activities that pass data and data persistence. I took platform-specific UI/UX differences into account for each version.

Key Features

  • Data Persistence
  • Multiple Views/Activites
  • TableViews/ListViews
  • Adaptable Layout
  • Custom Class for Data

DRIVING SUCKS!

Interactive Web Narrative

An interactive web narrative that points out the downsides of driving and provides alternatives.

Check it out!

Driving Sucks is a project that I created to shed light on the negative impacts of driving, both on the driver as well as on the community.

The narrative animates based on the users scroll position and first provides some arguments against driving before providing some alternatives.

At the end there is a tool that utilizes an API called Targomo to show how far a user can travel by foot, bike, or bus within a given timeframe. The point of this tool is to reveal the distance the user can go without a car in a relatively small amount of time.

Key Features

  • Interactivity
  • API usage
  • SVG animations

THE FINALS COUNTDOWN

A React Web App

A React app to track and plan for final exams.

Check it out!

The Finals Countdown is a React web app that I developed with team of three. The inspiration behind the product the desire to compartmentalize and prioritize studying based on how soon a final exam will be for a given course.

Users can input when their final exam will be and then add specific to-do items to each final. Our app then begins a countdown to the finals so that the user can keep track of what studying should be done first.

We used Firebase to persist the user's data and the application was built entirely with reusable React components. Our team used GitHub for version control and collaboration, and adhered to the Agile development process to increase efficiency.

Key Features

  • Built with React
  • Firebase
  • Agile development
  • Group collaboration via GitHub
